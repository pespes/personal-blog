# Figma ⇄ Code Sync Shim — Design

**Status:** Approved design (pre-implementation)
**Date:** 2026-06-28
**Repo:** `pespes/personal-blog` (Astro blog)
**Figma file:** `m8DPlUQk60FzGE02CnrhS2` — collection `Blog Tokens` (modes: Light, Dark)

## Problem

Every interaction with Figma currently requires a live MCP round-trip: slow, token-heavy,
and dependent on the Figma desktop app + auth. We want a lightweight local "shim" that caches
the shape of the design system so Claude Code can reason about and diff it **without** hitting
Figma each time, and sync only the deltas. The shim should also make
codebase ⇄ Claude Code ⇄ Figma syncing efficient and reviewable.

## Decisions (locked)

| Decision | Choice | Rationale |
| --- | --- | --- |
| Format | **Structured-first**, markdown generated | Precise round-trip data (per-mode values, Figma ids, variant schema) needs structured JSON; prose is lossy. |
| Conflict default | **Figma wins** | Design-led workflow. Figma→code auto-applies; code drift is flagged. |
| v1 scope | **Tokens + component manifest** | Tokens round-trip cleanly; components tracked as metadata/drift (no codegen). |
| Trigger | **`/figma-sync` project command** (Claude-run via MCP) | On Pro, Figma variables are only readable via the Plugin API (MCP) — the REST Variables API is Enterprise-only, so a standalone CI script cannot read tokens. |
| Token codegen | **A — marked-region regeneration in `global.css`** | Lightest; no new deps/files; keeps `global.css` the single readable home, with only token values machine-owned. |

## Constraints

- **Pro plan:** Figma variables are readable only through the Plugin API (`use_figma` via the MCP).
  REST Variables API is Enterprise-only. Therefore the sync engine must run inside Claude Code
  with the Figma desktop app open and the MCP authed. CI/CLI automation is out of scope for v1.
- Sync is **non-destructive to hand-written code**: only the marked CSS region and the generated
  JSON/markdown artifacts are written.

## Architecture

### Artifacts (the shim)

```
design/
  tokens.json       # canonical DTCG tokens (pulled from Figma variables)
  components.json   # component manifest (figmaKey <-> .astro <-> props/variants/tokens)
  sync-state.json   # per-entity hashes + lastSync (the ledger)
  DESIGN.md         # GENERATED, read-only overview (what Claude/humans read)
src/styles/global.css  # token CSS regenerated between markers (Figma wins)
```

### `tokens.json` (DTCG)

Canonical token record. Each token carries `$type`, `$value` (Light = default mode), and
`$extensions` with the Figma `variableId`, per-mode values, the `css` var name, and scopes.

```json
{
  "accent": {
    "$type": "color",
    "$value": "#006cac",
    "$extensions": {
      "com.figma": { "variableId": "VariableID:5:5", "collection": "Blog Tokens" },
      "mode": { "dark": "#5db1e8" },
      "css": "var(--accent)",
      "scopes": ["TEXT_FILL", "STROKE_COLOR", "FRAME_FILL", "SHAPE_FILL"]
    }
  }
}
```

### `components.json` (manifest — metadata only, no codegen)

One entry per component:

```json
{
  "Tag": {
    "figmaKey": "13:16",
    "type": "COMPONENT_SET",
    "source": "src/components/Tag.astro",
    "variants": { "Size": ["Large", "Small"] },
    "props": { "Label": "TEXT" },
    "boundTokens": ["foreground", "accent", "border"],
    "description": "Tag pill link…"
  }
}
```

### `sync-state.json` (the ledger)

Per entity: `figmaHash`, `codeHash`, `lastSync` (timestamp + commit), `status`. Enables cheap
diffing and drift detection.

```json
{
  "tokens.accent": { "figmaHash": "ab12…", "codeHash": "cd34…", "lastSync": "2026-06-28T10:00:00Z@<commit>", "status": "in-sync" },
  "components.Tag": { "figmaHash": "ef56…", "codeHash": "0789…", "lastSync": "2026-06-28T10:00:00Z@<commit>", "status": "code-drift" }
}
```

### `DESIGN.md` (generated, read-only)

Regenerated each sync from the JSON: token table, component map, drift summary, last-sync line.
**This is the everyday efficiency payoff** — Claude reads this local file for context during
normal codegen instead of round-tripping to Figma.

### `global.css` token target (option A)

Sync rewrites only the declarations between markers; everything else is untouched:

```css
/* figma-tokens:start — generated by /figma-sync, do not edit by hand */
:root, html[data-theme="light"] { --background:#fdfdfd; --foreground:#282728; --accent:#006cac; --muted:#e6e6e6; --border:#ece9e9; }
html[data-theme="dark"] { --background:#1c1917; --foreground:#ffffff; --accent:#5db1e8; --muted:#292524; --border:#57534e; }
/* figma-tokens:end */
```

## The `/figma-sync` command

Repo-local project command in `.claude/commands/figma-sync.md`, backed by a short skill.
Claude executes it using the MCP + file edits.

**Flags:** `--check` (drift report, no writes), `--dry-run` (show planned changes, no writes),
`--tokens-only`.

**Default behavior:** apply token changes (Figma wins) and print the diff; component/code drift
is report-only.

### Flow

1. **Preflight** — confirm Figma MCP connected + authed (`whoami`); abort cleanly if not (no partial writes).
2. **Pull** — via MCP (reuse `figma-design-system-inventory`): `Blog Tokens` collection
   (variables, modes, values, ids, scopes, codeSyntax) + component-set metadata
   (keys, variant props, descriptions, bound tokens).
3. **Normalize** — build candidate `tokens.json` + `components.json`.
4. **Diff** — compare candidate vs current canonical + `sync-state` hashes; classify each entity:
   *in-sync / figma-changed / code-drift / conflict*.
5. **Apply (Figma wins)** — tokens: regenerate the marked region in `global.css` + write
   `tokens.json`. Components: update `components.json` only (never touches `.astro`).
6. **Regenerate** `DESIGN.md`; update `sync-state.json` (hashes, timestamp, commit).
7. **Report** — applied changes + a **Drift/Action list** (code drift, unmapped components).

### Drift detection

`figmaHash` (normalized Figma data per entity) vs `codeHash` (the marked CSS block for tokens;
the prop signature/source path for components), each compared to the last-synced hashes in
`sync-state.json`:

- **Figma changed only** → apply.
- **Code changed only** → flag "code drift" (report; user decides to overwrite or push back).
- **Both changed** → conflict; Figma wins by default, flagged prominently.

### Edge cases

- New token in Figma → added to `tokens.json` + inserted in the CSS marked block.
- Token removed in Figma → removed from CSS; if `grep` finds refs in `src/`, **flag** (don't silently break).
- Per-mode value differences → both modes written (Light `:root`, dark block).
- Renamed token (stable `figmaId`, new name) → detected by id; var renamed + code refs flagged.
- Component in Figma with no matching `.astro` (or vice versa) → listed as **unmapped**; `source`
  mapping stored once in `components.json`.
- Figma unreachable (no desktop/MCP) → abort, no writes.

## Components & boundaries

- **Pull/normalize** (Figma → candidate JSON): depends on the MCP; reuses `figma-design-system-inventory`.
  Input: Figma file/collection. Output: candidate `tokens.json` + `components.json`.
- **Diff/classify** (pure function): inputs candidate + current canonical + `sync-state`;
  output: per-entity status list. No side effects — independently testable.
- **Apply** (writes): inputs classified diff; writes `global.css` marked region, `tokens.json`,
  `components.json`. Only touches marked region + artifacts.
- **Render** (`DESIGN.md` generator): pure function of the canonical JSON.
- **Command** (`/figma-sync`): orchestrates the above + reporting; owns flags and preflight.

Each unit has one purpose and a defined interface; diff/classify and render are pure and testable
without Figma.

## Non-goals (v1)

- Generating `.astro` component code.
- Syncing page / breakpoint mockups.
- CI automation (Pro-plan REST limit).
- Collections beyond `Blog Tokens` / themes beyond Light + Dark.

## Success criteria

- Running `/figma-sync` with no changes reports "in sync" and writes nothing new (idempotent).
- A color edit in Figma, after sync, updates `tokens.json`, the `global.css` marked region (both
  modes), and `DESIGN.md` — with hand-written CSS untouched.
- A hand edit to a token in `global.css` is reported as **code drift** on the next sync (not
  silently overwritten without notice).
- Between syncs, Claude answers token/component questions from `DESIGN.md` + `tokens.json` with
  no MCP call.
- `--check` exits with a drift summary and zero writes.

## Upgrade paths (future)

- **Option B/C codegen** (separate `tokens.css` or Style Dictionary) if a token build pipeline is added.
- **CLI + CI** if the workspace moves to a Figma Enterprise plan (unlocks REST Variables API).
- **Code → Figma push** for reconciling code drift back into Figma (currently manual/report-only).
